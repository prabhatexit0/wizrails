## Codebase Patterns
- Frontend is in `frontend/` — Vite + React + TypeScript
- WASM engine is in `wasm-engine/` — Rust with wasm-pack, target=web
- Build with `make build` (builds WASM first, then frontend)
- Dev server with `make dev` (builds WASM first, then starts Vite)
- Zustand store is at `frontend/src/store.ts`
- Canvas game components go in `frontend/src/components/`
- App.tsx is the main entry — React overlays on top of Canvas
- WASM imported from `../../wasm-engine/pkg/wasm_engine.js` (relative to frontend/src)
- Vite config at `frontend/vite.config.ts` has `server.fs.allow` for WASM pkg access
- Full design doc at `prompt.md` in project root — READ THIS for route/scenery details
- Pixel art canvas: render at 480x270, scale up with CSS `image-rendering: pixelated`
- Use requestAnimationFrame for game loop, React for UI overlays only
- Engine classes in `frontend/src/engine/`: GameLoop, ParallaxRenderer, TrainRenderer, PhysicsEngine, SignalEngine, EventEngine, StationRenderer, InteriorRenderer, AudioEngine, NPCEngine
- Screens (menu/results/accident/tutorial/pause) in `frontend/src/components/Screens.tsx`
- HUD control panel in `frontend/src/components/HUD.tsx`
- GameCanvas orchestrates all engine classes in `frontend/src/components/GameCanvas.tsx`
- WASM exports: check_signal, calculate_eta, punctuality_score, journey_score, calculate_rating
- TypeScript strict mode with verbatimModuleSyntax — use `import type` for type-only imports
